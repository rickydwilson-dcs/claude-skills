# Investigation Procedures

Detailed procedures for quality, safety, and process investigations.

## Quality Issue Investigation Guide

### Scope
Quality issues including nonconformances, out-of-specification results, customer complaints, and internal quality findings.

### Investigation Process

#### Phase 1: Initial Assessment (Day 1)
**Objectives:**
- Contain immediate impact
- Assess severity and scope
- Determine investigation priority

**Activities:**
1. **Document Initial Finding**
   - Date/time of discovery
   - Description of issue
   - Immediate impact assessment
   - Product/process affected

2. **Containment Actions**
   - Quarantine affected product
   - Stop affected processes if needed
   - Prevent further distribution
   - Document containment actions

3. **Severity Assessment**
   - Patient safety impact
   - Regulatory notification required?
   - Product recall potential
   - Scope of affected units

4. **Investigation Team Assignment**
   - Investigation lead designation
   - Subject matter experts identified
   - Resource allocation
   - Timeline establishment

#### Phase 2: Investigation Planning (Days 2-3)
**Objectives:**
- Develop investigation approach
- Identify information needs
- Establish investigation scope

**Activities:**
1. **Scope Definition**
   - Investigation boundaries
   - Time period under investigation
   - Products/processes included
   - Exclusions and rationale

2. **Investigation Plan Development**
   - RCA methodology selection
   - Data collection requirements
   - Interview list
   - Evidence preservation needs
   - Timeline and milestones

3. **Resource Preparation**
   - Access to systems and records
   - Investigation workspace setup
   - Tools and templates assembled
   - Team briefing conducted

#### Phase 3: Data Collection (Days 4-10)
**Objectives:**
- Gather comprehensive factual evidence
- Interview relevant personnel
- Review documentation and records

**Evidence Collection:**
1. **Physical Evidence**
   - Failed products or components
   - Environmental samples
   - Equipment records
   - Calibration certificates

2. **Documentary Evidence**
   - Batch/lot records
   - Work instructions and procedures
   - Training records
   - Change control records
   - Maintenance logs
   - Previous CAPA records

3. **Testimonial Evidence**
   - Operator interviews
   - Supervisor statements
   - Witness observations
   - Expert consultations

4. **Digital Evidence**
   - System logs and audit trails
   - Electronic batch records
   - Automated monitoring data
   - Email communications

**Interview Protocol:**
- Prepare questions in advance
- Conduct in neutral location
- Focus on facts, not blame
- Document responses accurately
- Obtain signature/confirmation
- Maintain confidentiality

#### Phase 4: Analysis (Days 11-15)
**Objectives:**
- Analyze collected evidence
- Identify root causes
- Verify conclusions

**Analysis Activities:**
1. **Data Organization and Review**
   - Timeline construction
   - Evidence mapping
   - Pattern identification
   - Gap analysis

2. **Root Cause Analysis**
   - Apply selected RCA methodology
   - Identify immediate causes
   - Identify contributing factors
   - Determine root causes
   - Document analysis process

3. **Verification Activities**
   - Test root cause hypotheses
   - Seek contradictory evidence
   - Validate with subject matter experts
   - Assess alternative explanations
   - Document verification results

4. **Risk Assessment Update**
   - Update risk analysis if needed
   - Assess residual risk
   - Determine risk acceptability
   - Document risk decisions

#### Phase 5: Corrective Action Development (Days 16-20)
**Objectives:**
- Develop effective corrective actions
- Plan implementation approach
- Establish verification methods

**Action Development:**
1. **Immediate Corrective Actions**
   - Short-term containment
   - Prevent immediate recurrence
   - Document actions taken

2. **Long-term Corrective Actions**
   - Address root causes
   - Prevent systemic recurrence
   - Sustainable solutions
   - Hierarchy of controls applied

3. **Preventive Actions**
   - Identify similar potential issues
   - Extend actions to related areas
   - System-wide improvements
   - Lessons learned application

4. **Implementation Planning**
   - Action items with owners
   - Timeline and milestones
   - Resource requirements
   - Success criteria definition
   - Communication plan

#### Phase 6: Documentation (Days 21-25)
**Objectives:**
- Complete investigation report
- Document all findings and actions
- Obtain approvals

**Report Contents:**
- Executive summary
- Investigation scope and methodology
- Findings and evidence
- Root cause analysis
- Corrective and preventive actions
- Implementation plan
- Effectiveness verification plan
- Appendices with supporting evidence

**Approvals Required:**
- Investigation lead
- Quality manager
- Department head (affected area)
- Regulatory affairs (if applicable)

#### Phase 7: Implementation (Ongoing)
**Objectives:**
- Execute corrective actions
- Monitor progress
- Adjust as needed

**Implementation Activities:**
- Action item tracking
- Progress monitoring
- Milestone verification
- Issue resolution
- Status reporting
- Documentation updates

#### Phase 8: Effectiveness Verification (3-12 months post-implementation)
**Objectives:**
- Verify action effectiveness
- Confirm sustained improvement
- Close CAPA or escalate

**Verification Methods:**
- Data analysis (metrics, trends)
- Process monitoring
- Audit findings
- Recurrence tracking
- Customer feedback review

**Closure Criteria:**
- All actions completed
- Effectiveness verified
- No recurrence observed
- Documentation complete
- Approvals obtained

## Safety Issue Investigation Guide

### Scope
Safety incidents including adverse events, near-misses, safety complaints, and potential safety issues.

### Unique Safety Considerations

1. **Regulatory Notification**
   - FDA MedWatch reporting (if applicable)
   - MDR reporting to EU authorities
   - Timeline compliance (critical)
   - Notification templates usage

2. **Patient Safety Priority**
   - Immediate hazard assessment
   - Field action evaluation (recall/removal)
   - Healthcare provider notification
   - Patient communication plan

3. **Safety Risk Assessment**
   - Harm severity evaluation
   - Probability of occurrence
   - Risk acceptability determination
   - Benefit-risk analysis update

4. **Cross-functional Team**
   - Regulatory affairs involvement
   - Medical affairs/clinical expertise
   - Risk management integration
   - Legal counsel (if needed)

### Safety Investigation Timeline
- **Day 1:** Initial safety assessment, containment
- **Days 2-5:** Preliminary investigation, regulatory notification
- **Days 6-15:** Full investigation, root cause analysis
- **Days 16-30:** Corrective action planning, regulatory updates
- **Ongoing:** Implementation, effectiveness verification, follow-up reporting

### Safety Documentation Requirements
All safety investigation documentation plus:
- Regulatory notification records
- Safety risk assessment
- Benefit-risk analysis
- Field action decision documentation
- Healthcare provider communications
- Patient notification records (if applicable)

## Process Issue Investigation Guide

### Scope
Process deviations, process capability issues, validation failures, and process performance problems.

### Process Investigation Approach

#### 1. Process Understanding
**Objectives:**
- Define process boundaries
- Identify process inputs/outputs
- Document process flow
- Establish baseline performance

**Activities:**
- Process mapping (flowchart)
- Input/output identification
- Critical process parameters (CPPs)
- Key quality attributes (KQAs)
- Process capability review

#### 2. Deviation Analysis
**Objectives:**
- Characterize process deviation
- Assess impact on product quality
- Determine investigation scope

**Activities:**
- Deviation description
- Timeline reconstruction
- Affected batches/lots identification
- Impact assessment (product quality)
- Investigation scope determination

#### 3. Process Data Analysis
**Objectives:**
- Analyze process performance data
- Identify process variations
- Correlate to quality issues

**Data Sources:**
- Process monitoring data
- Statistical process control (SPC) charts
- Batch records
- Equipment logs
- Environmental monitoring
- In-process testing results

**Analysis Methods:**
- Trend analysis
- Statistical analysis (mean, std dev, Cpk)
- Control chart analysis
- Correlation analysis
- Design of experiments (if appropriate)

#### 4. Root Cause Investigation
**Focus Areas:**
- Process design adequacy
- Process control effectiveness
- Equipment performance
- Material variability
- Environmental factors
- Human factors

**Common Process Root Causes:**
- Inadequate process validation
- Insufficient process monitoring
- Equipment malfunction or drift
- Material specification issues
- Environmental control inadequate
- Procedure inadequacy or ambiguity
- Training gaps

#### 5. Process Improvement Actions
**Action Types:**
1. **Process Validation Updates**
   - Revalidation if needed
   - Validation protocol updates
   - Acceptance criteria revision

2. **Process Control Enhancements**
   - Additional monitoring points
   - Control limits tightening
   - Automated controls implementation

3. **Process Design Modifications**
   - Process parameter optimization
   - Equipment upgrades
   - Technology improvements

4. **Documentation Updates**
   - Procedure revisions
   - Work instruction clarification
   - Process specifications update

#### 6. Process Requalification
**When Required:**
- Process changes implemented
- Validation invalidated
- Regulatory requirement
- Risk assessment indicates need

**Requalification Scope:**
- Affected process steps only, or
- Full process revalidation

### Process Investigation Documentation

**Required Documentation:**
- Process investigation report
- Process data analysis
- Root cause analysis
- Process validation impact assessment
- Corrective action plan
- Revalidation protocol (if needed)
- Change control linkage

## Investigation Quality Checklist

### Problem Definition Quality
- [ ] Problem clearly stated and specific
- [ ] Scope well-defined with boundaries
- [ ] Impact accurately assessed
- [ ] Severity appropriately determined
- [ ] Investigation priority justified

### Data Collection Quality
- [ ] Evidence comprehensive and complete
- [ ] Multiple evidence types collected
- [ ] Evidence properly preserved
- [ ] Chain of custody maintained
- [ ] Interviews conducted appropriately
- [ ] Documentation reviewed thoroughly

### Analysis Quality
- [ ] Appropriate RCA methodology selected
- [ ] Analysis systematic and thorough
- [ ] Root cause(s) clearly identified
- [ ] Alternative explanations considered
- [ ] Conclusions verified with evidence
- [ ] Risk assessment updated

### Corrective Action Quality
- [ ] Actions address root causes
- [ ] Actions prevent recurrence
- [ ] Actions feasible to implement
- [ ] Actions appropriately prioritized
- [ ] Implementation plan detailed
- [ ] Responsible parties assigned
- [ ] Timeline realistic and clear
- [ ] Resources identified

### Documentation Quality
- [ ] Investigation report complete
- [ ] Evidence included/referenced
- [ ] Analysis documented clearly
- [ ] Conclusions well-supported
- [ ] Actions clearly described
- [ ] Approvals obtained
- [ ] Traceability maintained

### Effectiveness Verification Quality
- [ ] Verification plan defined
- [ ] Verification methods appropriate
- [ ] Success criteria clear and measurable
- [ ] Verification timeline established
- [ ] Results documented
- [ ] Conclusions supported by data

## Common Investigation Mistakes to Avoid

**Rushing to Conclusions:**
- Accepting first plausible explanation
- Insufficient data collection
- Inadequate analysis
- Skipping verification steps

**Weak Root Cause Analysis:**
- Stopping at symptoms
- Blaming individuals vs. systems
- Single-factor analysis of complex problems
- No verification of root cause

**Incomplete Investigations:**
- Missing key evidence
- Limited stakeholder involvement
- Inadequate scope definition
- Poor documentation

**Ineffective Corrective Actions:**
- Treating symptoms not causes
- Over-reliance on administrative controls
- No preventive action component
- Unclear implementation plans
- Missing effectiveness verification

**Documentation Deficiencies:**
- Incomplete investigation records
- Missing evidence
- Unclear rationale for decisions
- No traceability
- Poor report quality
